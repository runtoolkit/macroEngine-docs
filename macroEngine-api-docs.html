API Reference

A high-performance scripting and macro execution framework for Minecraft Java Edition datapacks. Provides fiber scheduling, modular APIs, and multi-version overlay support out of the box.

🗄
Storage Convention
All modules communicate via three canonical storage namespaces. Never write directly to macro:engine — it is internal.
macro:input
Write call arguments here before invoking a module function. Cleared by the callee after reading.
macro:output
Return values are written here by the callee. Read immediately — next call will overwrite.
macro:engine
Internal runtime state (fiber stack, scheduler heap, counters). Read-only for user code.
input_push / input_pop
Use these helpers to safely nest module calls without clobbering macro:input.
⚠ Nesting rule: Always call macro:io/input_push before a nested module invocation and macro:io/input_pop after, otherwise the outer call's input data will be corrupted.
⚙
Macro Syntax Rules
Rules that apply to every .mcfunction file that uses AME macros.
✔
Macro line rules
• Every macro line must start with $
• Every macro line must contain at least one $(variable)
• Non-macro lines inside a macro function are executed literally — do not use $(…) in them
• Scoreboard fake-player names use #name prefix (not $name) in non-macro lines
ℹ
pack.mcmeta — supported_formats must always be {"min_inclusive":X,"max_inclusive":Y}. Array syntax [X,Y] is invalid on 1.21.10+ servers.
.mcfunction · macro example
# Set input, then call a module function
data modify storage macro:input target set value "my_target"
function macro:scheduler/run_after

# A macro function that reads $(delay) from storage
$schedule function macro:mypack/callback $(delay)t append

macro:scheduler
Fiber-based task scheduling. Defers function execution by a tick count without blocking the main tick. Supports repeating fibers and cancellation by ID.
FN macro:scheduler/run_after Schedule a function to run after N ticks ▾
Input (macro:input)
FieldTypeDescription
delayintTicks to wait before executionrequired
functionstringNamespaced function path to callrequired
idstringUnique fiber ID (for cancellation)optional
Output (macro:output)
FieldTypeDescription
fiber_idstringAssigned fiber ID (auto-generated if not supplied)
Example
.mcfunction
data modify storage macro:input delay set value 40
data modify storage macro:input function set value "mypack:on_expire"
data modify storage macro:input id set value "my_timer_fiber"
function macro:scheduler/run_after
FN macro:scheduler/repeat Run a function every N ticks indefinitely ▾
Input (macro:input)
FieldTypeDescription

macro:io
Input/output stack helpers. Provides input_push / input_pop for safe nested calls, plus convenience read/write wrappers.
FNmacro:io/input_pushSave current macro:input to the stack▾
✔Call before any nested module invocation. Pairs with macro:io/input_pop.
.mcfunction · nested call pattern
function macro:io/input_push
data modify storage macro:input name set value "inner_queue"
function macro:queue/pop
function macro:io/input_pop
FNmacro:io/input_popRestore macro:input from stack▾
⚠Each input_push must have exactly one matching input_pop. Mismatched calls corrupt the stack.
FNmacro:io/clear_inputWipe all fields in macro:input▾
ℹUseful before building a fresh input compound to avoid stale fields from a previous call.
